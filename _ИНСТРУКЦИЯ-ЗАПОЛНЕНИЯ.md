# Инструкция по заполнению направления

> Универсальный промт для Claude. Копируй и вставляй в новое окно, заменив `[НАПРАВЛЕНИЕ]` на нужное.

---

## Промт для заполнения направления

```
Заполни направление [НАПРАВЛЕНИЕ] по стандарту проекта "Виртуальная Школа Ремонта".

## Контекст проекта

Прочитай эти файлы для понимания структуры:
1. /CLAUDE.md — архитектура проекта
2. /направления/реестр-направлений.md — список всех направлений и связей
3. /технологии/шаблон-технологии.md — шаблон технологии
4. /направления/плитка/D12-плитка.md — ЭТАЛОН заполненного направления
5. /технологии/плитка/T-PL-001-укладка-керамогранита-на-пол.md — ЭТАЛОН технологии
6. /база-данных/материалы/плитка/мат-плитка.md — ЭТАЛОН базы материалов

## Что нужно сделать

### 1. Обновить файл направления
Файл: /направления/[папка-направления]/D[XX]-[название].md

Структура (по эталону плитки):
- YAML frontmatter (id, name, type, stages, status)
- Описание направления
- Таблица "Процессы и технологии" с ID технологий
- Порядок работ (схема последовательности)
- Требования к рабочему (таблица навыков)
- Материалы (таблица со ссылками на базу)
- Инструменты (таблица)
- Связанные направления (с указанием ДО/ПОСЛЕ)
- База данных (ссылки)
- Статистика направления

### 2. Создать первую технологию
Файл: /технологии/[папка]/T-XX-001-название.md

По шаблону /технологии/шаблон-технологии.md:
- YAML frontmatter
- Когда применять / Когда НЕ применять
- Преимущества / Недостатки
- Материалы и инструменты (таблицы)
- Подготовка (чек-лист)
- Пошаговая инструкция (5-10 шагов)
- Чек-лист качества (критические/важные/желательные)
- Приёмка работы (допуски)
- Техника безопасности
- FAQ (3-5 вопросов)
- Связанные технологии
- Вопросы для теста (5 вопросов разных типов)

### 3. Создать базу материалов
Файл: /база-данных/материалы/[направление]/мат-[направление].md

Структура:
- Категории материалов
- Классификация (если есть стандарты)
- Рекомендуемые бренды/продукты
- Расход материалов
- Связанные документы

### 4. Обновить реестр технологий
Файл: /технологии/реестр-технологий.md

Добавить секцию направления в "Список технологий"

## Правила заполнения

1. **Реалистичность** — только проверенная информация из строительной практики
2. **Конкретика** — точные цифры, допуски, расходы (не "примерно", а "3-5 кг/м²")
3. **Связи** — все направления связаны между собой (ДО/ПОСЛЕ)
4. **ID система** — соблюдать формат T-XX-NNN
5. **Статусы** — новые технологии = draft

## После заполнения

Проверь:
- [ ] Все ссылки работают
- [ ] ID технологий уникальны
- [ ] Связи между направлениями корректны (ДО/ПОСЛЕ)
- [ ] Цифры реалистичны (расход, допуски, время)
- [ ] Нет дублирования информации
- [ ] Чек-листы конкретные (как проверить)
- [ ] Тестовые вопросы имеют правильные ответы
```

## Быстрый старт

Для заполнения направления [НАЗВАНИЕ] выполни:

1. `mkdir -p технологии/[папка]`
2. `mkdir -p база-данных/материалы/[направление]`
3. Обнови направления/[папка]/D[XX]-[название].md
4. Создай технологии/[папка]/T-XX-001-название.md
5. Создай база-данных/материалы/[направление]/мат-[направление].md
6. Обнови технологии/реестр-технологий.md

## Пример промта для конкретного направления

```
Заполни направление ЭЛЕКТРИКА (D02) по стандарту проекта.

Прочитай эталоны:
- /направления/плитка/D12-плитка.md
- /технологии/плитка/T-PL-001-укладка-керамогранита-на-пол.md
- /база-данных/материалы/плитка/мат-плитка.md
- /технологии/шаблон-технологии.md

Создай:
1. Обнови /направления/электрика/D02-электрика.md по структуре эталона
2. Создай /технологии/электрика/T-EL-001-штробление-штроборезом.md
3. Создай /база-данных/материалы/электрика/мат-электрика.md (объедини черновая.md + чистовая.md)
4. Добавь электрику в /технологии/реестр-технологий.md

Префикс технологий: T-EL
Этапы: черновая + чистовая
```

## Чек-лист качества направления

После заполнения проверить:

### Структура
- [ ] D[XX]-[название].md обновлён по эталону плитки
- [ ] Создана минимум 1 технология
- [ ] Создана база материалов
- [ ] Обновлён реестр технологий

### Содержание
- [ ] Процессы соответствуют реальной практике
- [ ] Порядок работ логичен
- [ ] Связи ДО/ПОСЛЕ корректны
- [ ] Материалы актуальны (бренды существуют)
- [ ] Расходы реалистичны

### Технология
- [ ] Шаги последовательны и полны
- [ ] Чек-листы конкретные (как проверить)
- [ ] Допуски по СНиП/ГОСТ
- [ ] ТБ соответствует работам
- [ ] Тестовые вопросы корректны

### Ссылки
- [ ] Все внутренние ссылки работают
- [ ] ID технологий уникальны в проекте
- [ ] Нет битых ссылок на базу данных
